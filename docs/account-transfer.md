# 口座振替

自動引き落としを決済代行会社が行うこと

決済代行会社へ、契約者（サービス提供者）がサービス料金の引き落としを依頼する

依頼された決済代行会社は、契約者から連携された 銀行口座情報を利用し、金融機関へとサービス料金の引き落としを依頼する

各金融機関がサービス料金の引き落としを実行し、依頼結果を決済代行会社へと連絡する

決済代行会社は、各金融機関から受け取った実行結果を集め、契約者へ引き落とし結果を連携する

起こりうる事態

*   口座情報の誤り
*   残高不足

## G4 と、`Rac.VOne.AccountTransfer` での相違点

*   `Rac.VOne.Client.Screen` の実装を `Rac.VOne.AccountTransfer` へ分離
*   `System.IO` 関連の処理を除外 （外で実装してね）
*   `Matcher` の削除 すべて `Reader` へ実装をまとめた
*   `ConcreteRecord` の 検証処理緩和
    *   Trailer/End Record の実装を削除（どこにも何にも利用していない）
    *   Header/Data Record の fields length の検証処理を修正
        *   取得が必要な 最終 field の インデックス以下であれば例外を投げる
*   `MatchingResult` 照合結果 という `Web.Models` で重複している class name を変更
    *   `AccountTransferSource` と名前を変え、 `Web.Models` 以下に配置
*   Db アクセス（WCF呼び出し）などをすべて `Func` デリゲートに変更
*   Screen に記述してあるビジネスロジックを `Rac.VOne.AccountTransfer` へ一本化
