detectChanges

  • detectChanges<T>(component: T): void
    • Defined in node_modules/@angular/core/src/render3/instructions.d.ts:549

    Synchronously perform change detection on a component (and possibly its sub-components).

    This function triggers change detection in a synchronous way on a component. There should be very little reason to call this function directly since a preferred way to do change detection is to markDirty the component and wait for the scheduler to call this method at some future point in time. This is because a single user action often results in many components being invalidated and calling change detection on each component synchronously would be inefficient. It is better to wait until all components are marked as dirty and then perform single change detection across all of the components

    Type parameters

    • T

    Parameters

    • component: T

      The component which the change detection should be performed on.

    Returns void

projectionDef

  • projectionDef(selectors?: CssSelectorList[], textSelectors?: string[]): void
    • Defined in node_modules/@angular/core/src/render3/instructions.d.ts:467

    Instruction to distribute projectable nodes among occurrences in a given template. It takes all the selectors from the entire component's template and decides where each projected node belongs (it re-distributes nodes among "buckets" where each "bucket" is backed by a selector).

    This function requires CSS selectors to be provided in 2 forms: parsed (by a compiler) and text, un-parsed form.

    The parsed form is needed for efficient matching of a node against a given CSS selector. The un-parsed, textual form is needed for support of the ngProjectAs attribute.

    Having a CSS selector in 2 different formats is not ideal, but alternatives have even more drawbacks:

    • having only a textual form would require runtime parsing of CSS selectors;
    • we can't have only a parsed as we can't re-construct textual form from it (as entered by a template author).

    Parameters

    • Optional selectors: CssSelectorList[]

      A collection of parsed CSS selectors

    • Optional textSelectors: string[]

    Returns void
